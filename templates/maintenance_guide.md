## 1. 知识库架构一览

```
my-wiki/                          # 知识库源文件（你编辑这里）
├── knowledge_index.md            # 三层索引（核心入口）
├── projects/                     # 项目知识
├── org/                          # 组织内部文档
├── methods/                      # 方法论
├── cases/                        # 案例库
└── ...

rag-data/                         # 向量数据库（自动生成，不要手动改）
└── wiki_vectors.json             # 核心向量数据库

scripts/                          # 工具脚本
├── config.yaml                   # 配置文件
├── embed_wiki.py                 # 向量化脚本（全量）
├── incremental_update.py         # 增量更新脚本（日常用这个）
├── agentic_rag.py                # RAG查询
└── rag_helper.py                 # 简单检索
```

**关键原则**：
- ✅ 你只编辑 `my-wiki/` 目录下的 `.md` 文件
- ❌ 不要手动修改 `rag-data/wiki_vectors.json`（由脚本自动维护）
- ✅ 每次编辑完文档后，更新 `knowledge_index.md` 索引
- ✅ 如果启用了RAG，编辑后运行增量更新脚本

## 2. 添加新文档

### 2.1 放到哪里？

按内容类型选择目录：

| 内容类型 | 放哪个目录 | 示例 |
|----------|-----------|------|
| 项目相关 | `projects/项目名/` | `projects/社区回收/计划.md` |
| 会议纪要 | `org/meetings/` | `org/meetings/2026-06-25_月度例会.md` |
| 制度流程 | `org/policies/` | `org/policies/志愿者管理.md` |
| 方法论 | `methods/` | `methods/社区动员方法.md` |
| 案例总结 | `cases/` | `cases/2026-06-20_义诊活动总结.md` |
| 日常工作 | `daily/YYYY-MM/` | `daily/2026-06/2026-06-25.md` |

### 2.2 文件命名规范

- 用中文，简洁描述内容
- 包含日期（格式：`YYYY-MM-DD_主题.md`）
- 示例：`2026-06-25_社区活动总结.md`

### 2.3 文档格式

每个文档头部应包含元信息：

```markdown
# [文档标题]

> **创建**: YYYY-MM-DD | **更新**: YYYY-MM-DD
> **维护者**: [姓名/角色]
> **状态**: 草稿 / 正式 / 归档

---

[正文内容]
```

### 2.4 更新索引

新增文档后，必须在 `knowledge_index.md` 中登记：
1. 第1层（目录索引）：添加到对应类别
2. 第2层（主题索引）：如果是新主题，添加主题条目
3. 第3层（提问场景索引）：如果涉及高频问题，添加场景条目

### 2.5 运行增量更新（如启用RAG）

```bash
cd scripts
python incremental_update.py
```

**预期输出**：
```
============================================================
增量向量化 v1.0
============================================================

文件扫描: 75 个 .md
  新增: 1
  修改: 0
  删除: 0

[处理] 2026-06-25_社区活动总结.md
  -> 8 chunks

============================================================
DONE!
  总chunks:    1689
  平均长度:    235 字符
  <100字符:    120 (7.1%)
============================================================
```

## 3. 修改文档

### 3.1 编辑文档

直接编辑 `my-wiki/` 目录下的 `.md` 文件（用任何编辑器：VS Code / Typora / 记事本）

### 3.2 运行增量更新

```bash
cd scripts
python incremental_update.py
```

脚本会自动检测哪些文件被修改了，只重新向量化这些文件。

### 3.3 更新索引

如果文档的主题/内容有重大变化，同步更新 `knowledge_index.md`。

## 4. 搜索知识库

### 4.1 命令行搜索（推荐）

```bash
cd scripts
python rag_helper.py --query "你的问题"
```

**示例**：
```bash
python rag_helper.py --query "志愿者管理制度"
python rag_helper.py --query "上次社区活动怎么做的"
python rag_helper.py --query "项目进展"
```

### 4.2 Agentic RAG 查询（智能判断）

```bash
cd scripts
python agentic_rag.py "你的问题"
```

Agentic RAG 会先判断是否需要检索知识库，再生成答案。

### 4.3 理解搜索结果

```
============================================================
查询: 志愿者管理
策略: vector
模型: nomic-embed-text
============================================================

【结果 1】相似度: 0.8245
文件: 志愿者管理.md
内容:
> 志愿者招募流程...
------------------------------------------------------------
```

- **相似度**越高越好（0-1，>0.7算很相关）
- **文件**告诉你这段内容来自哪个文档
- **内容**是chunk原文（可能不完整，只是片段）

### 4.4 如果搜索结果不理想

1. 换个问法（用同义词）
2. 检查向量数据库是否最新（运行 `python incremental_update.py`）
3. 看 [第5章](#5-故障排查)

## 5. 故障排查

### 问题1：脚本报错 `ModuleNotFoundError`

**原因**：脚本需要在 `scripts/` 目录下运行

**解决**：
```bash
cd scripts
python incremental_update.py
```

### 问题2：报错 `Connection refused`（Ollama没启动）

**原因**：向量化需要 Ollama 运行

**解决**：
1. 打开 Ollama 应用（系统托盘里找图标）
2. 或运行 `ollama serve`（命令行启动）
3. 验证：浏览器打开 `http://localhost:11434` 有响应

### 问题3：搜索结果不准确

**排查步骤**：

1. 检查向量数据库是否最新：
   ```bash
   cd scripts
   python incremental_update.py
   ```

2. 检查 chunk 质量：
   ```bash
   cd scripts
   python embed_wiki.py --stats
   ```

3. 如果平均长度 <150 字符，需要全量重建：
   ```bash
   cd scripts
   python incremental_update.py --full
   ```

### 问题4：向量数据库文件损坏

**解决**：全量重建

```bash
cd scripts
python incremental_update.py --full
```

**注意**：重建前先备份：
```bash
copy "rag-data\wiki_vectors.json" "rag-data\wiki_vectors.json.backup"
```

### 问题5：增量更新后文件数没变

**原因**：脚本只处理 `.md` 文件，其他格式不会处理

**解决**：确认文件是 `.md` 后缀。

## 6. 统计与质量检查

### 6.1 查看知识库统计

```bash
cd scripts
python embed_wiki.py --stats
```

或：
```bash
python rag_helper.py --stats
```

### 6.2 什么时候需要全量重建？

| 情况 | 操作 |
|------|------|
| 修改了切割策略 | 全量重建 |
| 换了嵌入模型 | 全量重建 |
| 平均chunk长度 <150字符 | 全量重建 |
| 只是日常修改文档 | 增量更新（不用重建）|

## 7. 维护节奏

| 频率 | 动作 | 耗时 |
|------|------|------|
| **随时** | 新增/修改文档后更新索引 + 增量更新 | 2分钟 |
| **每周** | 检查是否有孤立文件（未被索引） | 10分钟 |
| **每月** | 审核过时内容，归档或删除 | 30分钟 |
| **每季度** | 全量重建向量数据库，优化目录结构 | 1小时 |

## 8. 常见问题 FAQ

### Q1：不小心删了 wiki_vectors.json，怎么办？
**A**：全量重建：`python incremental_update.py --full`

### Q2：搜索很慢，正常吗？
**A**：正常。每次搜索要计算余弦相似度，约1-3秒。如果超过10秒，检查Ollama是否正常。

### Q3：可以手动编辑 wiki_vectors.json 吗？
**A**：❌ 不可以。这个文件由脚本自动生成。手动改会破坏结构。

### Q4：增量更新和全量重建有什么区别？
**A**：
- **增量更新**：只处理修改/新增的文件，快（几秒到1分钟）
- **全量重建**：重新处理所有文件，慢（取决于文件数量）
- **日常用增量，出问题才全量重建**

### Q5：Ollama 占用内存很高，可以关掉吗？
**A**：可以关，但关掉后增量更新和搜索都用不了。建议不用时关，需要时再开。
