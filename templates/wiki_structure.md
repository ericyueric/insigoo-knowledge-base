# 知识库目录结构模板

> 本文档说明知识库的标准目录结构，可按需调整。

---

## 标准目录结构

```
my-wiki/                          # 知识库根目录
├── knowledge_index.md            # 三层索引（核心入口，必须）
├── README.md                     # 知识库说明
│
├── projects/                     # 项目知识（按项目分目录）
│   ├── 项目A/
│   │   ├── README.md             # 项目概述
│   │   ├── 计划.md               # 项目计划
│   │   ├── 进展.md               # 进展追踪
│   │   └── 资料/                 # 项目相关资料
│   └── 项目B/
│       └── ...
│
├── org/                          # 组织内部文档
│   ├── intro.md                  # 组织简介
│   ├── meetings/                 # 会议纪要
│   │   └── YYYY-MM-DD_主题.md
│   ├── policies/                 # 制度流程
│   │   ├── 财务制度.md
│   │   └── 志愿者管理.md
│   └── members/                  # 成员档案
│       └── 姓名.md
│
├── methods/                      # 方法论/工具使用
│   ├── 活动组织方法.md
│   └── 工具使用指南.md
│
├── cases/                        # 案例库
│   ├── YYYY-MM-DD_案例名.md
│   └── ...
│
├── resources/                    # 外部资源
│   ├── 行业报告/
│   └── 参考资料/
│
├── daily/                        # 日常工作记录
│   ├── YYYY-MM/
│   │   └── YYYY-MM-DD.md
│   └── ...
│
└── archive/                      # 归档（过时但保留）
    └── ...
```

---

## 目录用途说明

| 目录 | 放什么 | 命名规范 |
|------|--------|----------|
| `projects/` | 每个项目一个子目录 | `projects/项目名/` |
| `org/meetings/` | 会议纪要 | `YYYY-MM-DD_主题.md` |
| `org/policies/` | 制度、流程、规范 | `主题.md` |
| `org/members/` | 成员信息 | `姓名.md` |
| `methods/` | 工作方法、工具教程 | `主题.md` |
| `cases/` | 实践案例 | `YYYY-MM-DD_案例名.md` |
| `resources/` | 外部资料 | 按类型分子目录 |
| `daily/` | 日志、周报 | `YYYY-MM/YYYY-MM-DD.md` |
| `archive/` | 过时文档 | 保持原结构 |

---

## 文件命名规范

### 通用规则

1. **用中文或英文**，简洁描述内容
2. **包含日期**（需要时间排序的文件）：`YYYY-MM-DD_主题.md`
3. **不用空格**，用下划线 `_` 或连字符 `-`
4. **不用特殊字符**：`/ \ : * ? " < > |`

### 示例

| 场景 | 命名 | 说明 |
|------|------|------|
| 会议纪要 | `2026-06-25_月度例会.md` | 日期+主题 |
| 项目文档 | `README.md` | 项目概述用README |
| 活动总结 | `2026-06-20_社区义诊总结.md` | 日期+主题 |
| 制度文档 | `志愿者管理制度.md` | 主题命名 |
| 方法论 | `社区动员方法.md` | 主题命名 |

## 何时创建新目录？

| 情况 | 操作 |
|------|------|
| 新项目启动 | 在 `projects/` 下创建项目子目录 |
| 新业务线 | 在根目录创建新主题目录 |
| 文档类型增多 | 在对应主题下创建子目录 |
| 目录文件>20个 | 考虑拆分子目录 |

**原则**：目录层级不超过3层，避免过度嵌套。

---

## 迁移现有文档

如果已有散落的文档，按以下步骤迁移：

1. **盘点**：列出所有现有文档，标注类型
2. **分类**：按上述目录结构归类
3. **重命名**：按命名规范重命名
4. **迁移**：移动到对应目录
5. **建索引**：在 `knowledge_index.md` 中登记
6. **验证**：检查链接是否有效

---

*可根据组织实际需求调整目录结构，但建议保持 projects/org/methods/cases/archive 五大基础目录。*
